Teddy's Tale — Django-проект сайта авторских мишек Тедди
Общее описание проекта.

Teddy's Tale — это полнофункциональный веб-сайт для мастера по созданию авторских мишек Тедди ручной работы. Проект реализован на Django и включает публичную часть сайта для клиентов и административную панель для управления всем контентом.

Основные характеристики
Фронтенд: адаптивный дизайн с мобильной версией

Бэкенд: Django 5.2+ с поддержкой SQLite (разработка) и PostgreSQL (продакшен)

Аутентификация: кастомная система входа в админ-панель

Управление контентом: гибкая система редактирования всех разделов сайта

Файловый менеджмент: загрузка, валидация и управление изображениями

Логирование: полный журнал изменений в админ-панели

Реализованные возможности (Django Backend):
1. Модели данных (Models)
   Созданы следующие модели для хранения и управления контентом:

PageSection – управление секциями сайта (meta, hero, shop, about, contacts)

SectionContent – гибкое хранение контента разного типа (текст, изображения, ссылки и др.)

ShopItem – товары с слотами (1–9), названием, описанием, ценой и изображением

UploadedImage – система управления загруженными изображениями

SiteSettings – настройки сайта с поддержкой разных типов данных

ChangeLog – журнал изменений для отслеживания действий в админ-панели

2. Административная панель (Admin Panel)
   Реализована полноценная CRUD-система с удобным интерфейсом:

Управление секциями (PageSection): включение/отключение, порядок, контент через инлайн-формы

Товары (ShopItem): редактирование названия, описания, цены, загрузка изображений с предпросмотром

Изображения (UploadedImage): просмотр, фильтрация, информация о файлах

Настройки (SiteSettings): управление ключами, значениями, категориями

Логи изменений (ChangeLog): только для чтения, с фильтрацией по дате и действию

3. Аутентификация и безопасность
   Кастомная форма входа с поддержкой сессий и cookies

Защита от CSRF- и XSS-атак

Валидация всех входных данных на стороне сервера

Хэширование паролей

4. Файловый менеджер
   Загрузка изображений через формы с валидацией формата и размера

Автоматическое переименование файлов

Хранение в директории media/

Поддержка отображения текущего изображения в админ-панели

5. Гибкая система контента
   Динамическое управление текстами, изображениями, ссылками через SectionContent

Поддержка типов: текст, многострочный текст, изображение, URL, телефон, email, координаты

Уникальные ключи для каждого элемента контента

6. Журналирование изменений
   Автоматическое логирование всех действий в админ-панели

Запись пользователя, IP, User Agent, старого и нового значения

Запрет на ручное редактирование логов

7. Интеграции
   Подготовка для интеграции с Яндекс.Картами через координаты в контенте

Динамическое обновление данных через админ-панель

Используемые технологии Django
Models: CharField, TextField, ImageField, ForeignKey, IntegerField с валидацией

Views: Class-based views для админки, Function-based views для публичной части

Forms: ModelForm, кастомные формы с валидацией

Templates: наследование шаблонов, динамические данные

Middleware: обработка запросов и ответов

Settings: раздельные настройки для разработки и продакшена

База данных ->
Разработка: SQLite3

Продакшен: PostgreSQL (подготовлено для миграции)

Миграции: автоматическое создание при изменении моделей

Установка и запуск:

1) Клонировать репозиторий

2) Установить зависимости: pip install -r requirements.txt

3) Выполнить миграции: python manage.py migrate

4) Создать суперпользователя: python manage.py createsuperuser

5) Запустить сервер: python manage.py runserver
