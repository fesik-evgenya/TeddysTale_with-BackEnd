Teddy's Tale - Django проект сайта авторских мишек Тедди

Общее описание проекта:

Teddy's Tale - это полнофункциональный веб-сайт для мастера по созданию авторских мишек Тедди ручной работы. Проект реализован на Django и включает в себя как публичную часть сайта для клиентов, так и административную панель для управления контентом.

Основные характеристики:

Фронтенд: Адаптивный дизайн с мобильной версией

Бэкенд: Django 3.2+ с использованием SQLite/PostgreSQL

Аутентификация: Кастомная система входа в админ-панель

Управление контентом: Динамическое редактирование всех разделов сайта

Файловый менеджмент: Загрузка и управление изображениями

Реализованные возможности (Django Backend) ->

1. Модели данных (Models)
   Созданы следующие модели для хранения контента:

MetaInfo - для хранения SEO-данных (title, keywords, description)

HeroSection - данные главной секции сайта

Product - модель товаров с полями: название, описание, цена, изображение

AboutSection - информация о мастере и преимущества

ContactInfo - контактные данные, координаты для карты

SocialLink - ссылки на социальные сети

User - кастомная модель пользователя для админ-панели

2. Административная панель (Admin Panel)
   Реализована полноценная CRUD-система для управления контентом:

Meta-информация:
Динамическое редактирование title, keywords, description

Валидация и сохранение SEO-данных

Главная секция (Hero):
Редактирование заголовка и описания

Загрузка и обновление главного изображения

Поддержка форматов: JPG, PNG, WEBP

Управление товарами (9 слотов):
Для каждого товара: название, описание, цена

Индивидуальная загрузка изображений для каждого товара

Система выбора слота для редактирования

Валидация размеров изображений (280px × 200px, до 1MB)

Секция "О мастере":
Редактирование заголовков и текстовых блоков

Управление списком преимуществ (4 пункта)

Загрузка фото мастера (480px × 400px, до 2MB)

Контакты:
Управление контактной информацией

Координаты для Яндекс.Карт (широта/долгота)

Редактирование ссылок на соцсети (VK, WhatsApp, Telegram)

3. Аутентификация и безопасность
   Кастомная форма входа с полями: логин, пароль, "запомнить меня"

Сессии и cookies для авторизации

Защита от CSRF-атак

Валидация входных данных

Безопасное хранение паролей (хэширование)

4. Файловый менеджер
   Загрузка изображений через форму

Валидация по формату и размеру

Автоматическое переименование файлов

Хранение в директории media/

Поддержка отображения текущего изображения

5. API и AJAX-взаимодействие
   Обновление текстовых данных без перезагрузки страницы

Асинхронная загрузка изображений

Обработка ответов сервера (успех/ошибка)

Динамическое обновление интерфейса

6. Интеграция с внешними сервисами
   Яндекс.Карты API для отображения местоположения

Динамическое обновление координат через админ-панель

7. Безопасность и валидация
   Проверка прав доступа к админ-панели

Валидация всех форм на стороне сервера

Защита от XSS-атак (экранирование данных)

Проверка MIME-типов загружаемых файлов

8. Панель навигации
   Умная прокрутка к выбранному разделу

Select-бокс для быстрого перехода между товарами

Интуитивно понятный интерфейс управления

9. Обработка ошибок
   Пользовательские сообщения об ошибках

Валидация на стороне клиента и сервера

Логирование действий администратора

-> Техническая реализация
Архитектура:

teddys_tale/
├── main/                      # Основное приложение
│   ├── models.py             # Все модели данных
│   ├── views.py              # Контроллеры для админки и сайта
│   ├── forms.py              # Формы для редактирования
│   ├── admin.py              # Настройка админ-панели Django
│   └── templates/            # HTML-шаблоны
├── static/                   # CSS, JavaScript, шрифты
├── media/                    # Загружаемые изображения
└── requirements.txt          # Зависимости проекта

Технологии Django:

Models: Использование полей: CharField, TextField, ImageField, DecimalField

Views: Class-based views для админки, Function-based views для публичной части

Forms: Кастомные формы с валидацией, ModelForm для CRUD

Templates: Наследование шаблонов, включение динамических данных

Static files: Организация CSS/JS, предзагрузка шрифтов

Middleware: Для обработки запросов и ответов

Settings: Раздельные настройки для разработки и продакшена

База данных:

Разработка: SQLite3
Продакшен: PostgreSQL (подготовлено для миграции)
Миграции: Автоматическое создание при изменении моделей
